Lock Lock Who's There:

Reverse Engineering a Smart Lock

University of Michigan

Winter 2016 Paper

The rise of the Internet of Things has prompted lock manufacturers to integrate connectivity into their products. These new smart locks promise to offer many convenience and security features, such as time limited keys, keyless entry, and audit logs. However, locks are commonly the primary line of defense for most homes and businesses, and the added complexity these systems introduce has the potential to introduce many serious vulnerabilities.

In the paper, we select a very popular smart lock and perform an in depth security audit of it’s firmware, authentication protocol, and mobile app. We present several practical attacks that can be performed against this device, including a backdoor that is able to provide on-demand surreptitious entry and a malicious mobile app that will steal the user’s credentials.

Data Integration in Real-Time

University of Michigan

Winter 2015 GitHub

The data integration in real-time (diirt) open source project aims to provide a platform to gather data from multiple sources (control systems, web sources, and databases) in real-time, aggregate them, and serve them either through UI or programmatic access.

I rewrote a diirt service library to allow user functions to be wrapped and executed either sychronously or asynchronously. Additionally, I refactored existing documentation that was provided both to developers and to users.

Graphene

University of Michigan

Fall 2013 — Winter 2014 GitHub

During my first year at University of Michigan, I participated in the Undergradaute Research Opportunity Program. I contributed to the open source Graphene project, a module of the the diirt project. Graphene is a real-time Java graphing library, used to display data from large physics experiments.

Another student and I developed sparkline and multi-line graph rendering capabilities, allowing rendering of large datasets as a sparkline graph or multi-line graph in under twenty milliseconds. I wrote a profiling suite to monitor the rendering efficiency of varying graph types with varying dataset sizes.

Assisted Video Editor

Winter 2016

Presentation

Assisted Video Editor (AVE) is a desktop application and companion iOS application for simplified video editing. The project's target audience was individuals with cerebral palsy. In particular, we targeted a student who only has fine motor control in one foot. The video editiing industry currently lacks software designed for ease of use. AVE provides creative powers to all individuals. The AVE application allows users to combine videos, photos, and audio together and share their content with the world.

AVE is a python desktop solution that imports photos, videos, and audio. A simplistic editor user interface allows the user to set an order for the imported content. The ordered content will be combined into a single video, with checkboxes allowing a user to upload the combined video onto social media services such as YouTube, Facebook, and Twitter. The editor also provides the option to automatically arrange media content, sychronizing the transition of clips with the beat of music.

The companion swift iOS solution is provided to improve ease of access. A user connects to an active instance of the desktop application, and can transfer photos and videos from their iOS device to the desktop application (which performs video processing). This solution design was implemented in order to allow multiple users on iOS devices to collaborate and combine their videos into one shared video.
